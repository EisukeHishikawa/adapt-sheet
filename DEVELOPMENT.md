# 開発ロードマップ

本プロジェクトは、エンジニアが保守しやすいHTML/CSSを生成する機能と、リアルタイムプレビューを見ながらHTML帳票を作成できるサイトの開発手順書です。
ClaudeCodeを活用し、**テスト駆動開発（TDD）**で最小限の機能から段階的に肉付けしていく「アジャイルアプローチ」で進めます。

また、AWS Lambdaのコールドスタート対策として、**「Doclingモデルのコンテナへの事前焼き込み」**および**「AWS Lambda Web Adapter」**の2つの技術をインフラのコア要件として組み込み、低コストかつ超高速なサーバーレス環境を実現します。

#### ⬛ ステップ 13: 構造化ログ基盤の導入（Python/FastAPIのログ設計）
> 【ステップ12の次に差し込み】以降のステップ（バックエンド分離・機能追加）が依存する横断的な観測基盤を先に整える。現状のバックエンドはログ出力が一切なく、`/api/render` のどの段階（JSON検証・Docling変換・AI生成）で失敗したかをサーバー側で追跡できない。Pythonの標準`logging`をベースに、リクエスト単位の相関ID（`request_id`）付き構造化ログ（JSON1行/レコード）とアクセスログのミドルウェアを導入し、後続のエラー設計（ステップ14）で返す相関IDと突き合わせられるようにする。**挿入理由**: バックエンド分離（新ステップ15）でプロセス/コンテナが増えるほどログの相関付けが重要になるため、分離より前に観測基盤を作る。標準ライブラリのみで実装し依存を増やさない（ADR-016）。
- [x] ⚙️ **設計:** `docs/decisions.md` にログ設計方針をADR-016として記録（標準`logging`ベースのJSON構造化ログ、`request_id`相関、ログレベル運用、APIキー等の機微情報を出力しない方針）
- [x] 🧪 **テストコード作成:** レスポンスに`X-Request-ID`ヘッダーが付与されること、リクエスト処理で構造化ログ（`request_id`・method・path・status・duration_ms）が出力されることを`caplog`で検証するテストを先に記述

#### ⬛ ステップ 14: API通信のエラー設計とフロント表示（構造化エラーレスポンス）
> 【ステップ13の次に差し込み】バックエンドで発生したエラーを、安全かつ一貫した形でフロント画面のメッセージに反映できるようにする。現状はバックエンドが`detail`文字列（技術的詳細を含みうる）を返し、フロントはHTTPステータスから静的文言に丸めるだけで、原因の粒度や追跡IDを画面に出せない。エラーレスポンスを`{"error": {"code", "message", "request_id"}}`の構造化エンベロープに統一し、`code`（機械可読な識別子）・`message`（ユーザー向けの安全な日本語文言）・`request_id`（ステップ13のログと相関）を返す。フロントはこの`message`を優先表示し、`request_id`を問い合わせ用に保持する。技術的詳細はサーバーログにのみ残し、レスポンスには漏らさない。**挿入理由**: ログ基盤（ステップ13）の相関IDを活かし、以降のUI刷新（ステップ16以降）や機能追加で増えるエラー経路をユーザーへ確実に伝える土台を先に固めるため（ADR-017）。
- [x] ⚙️ **設計:** `docs/decisions.md` にエラーレスポンス設計をADR-017として記録し、`docs/spec.md` 4章に構造化エラーボディのスキーマ・`X-Request-ID`を追記
- [x] 🧪 **テストコード作成（バック）:** 400/422/502/500の各エラーで`{"error":{code,message,request_id}}`形式のボディを返し、ボディの`request_id`が`X-Request-ID`ヘッダーと一致することを検証するテストを先に記述
- [x] 🧪 **テストコード作成（フロント）:** 構造化エラーボディの`message`を画面表示用に取り出せること、ボディが無い場合はステータス別の既定文言にフォールバックすることを検証するVitestテストを先に記述
- [x] **実装（バック）:** FastAPIの例外ハンドラで例外種別→`code`/安全な`message`/ステータスへマッピングし、構造化エラーボディを返す
- [x] **実装（フロント）:** `RenderApiError`が`code`/`message`/`request_id`を保持し、`sheetStore`がバックエンド提供の`message`を優先表示するよう拡張
