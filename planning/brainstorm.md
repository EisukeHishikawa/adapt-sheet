# 帳票作成AI支援プラットフォーム 構想メモ本ドキュメントは、プロジェクト発足時のコアアイデア、要件、および技術スタックのブレインストーミングをまとめたものである。開発の進行に伴い、技術的な改善点や実務的なアプローチを追記・最適化している。

---

## 🎯 プロダクトビジョン

> **「エンジニアが圧倒的に保守しやすいHTML帳票を、AIの力で美しく、素早く構築・管理する」**
> 既存のPDFをベースにしたモダンなHTML/CSS帳票の生成と、リアルタイムなプレビュー環境を統合した、開発者のための帳票作成支援SPA。

### 💡 コア開発思想
- **ドキュメント・ルール駆動:** 思想とルールを常に言語化し、Markdown（特に `CLAUDE.md`, `DEVELOPMENT.md`）に落とし込んでAI（ClaudeCode）との共通認識を保ちながら開発を進める。
- **インクリメンタル＆強固なTDD:** 一気に作らず、最小機能から「テスト駆動開発（TDD）」を徹底。テストコードを必ず実装の前に書き、確認と改善を繰り返してコードを育てる。
- **最後のアドオンとしてのインフラ/認証:** コアとなる価値（AI生成・プレビュー連動）のローカル完成度を最優先し、インフラ構築・セキュリティ・認証認可は後半に疎結合に組み込む。

---

## 🛠️ アーキテクチャ ＆ 技術スタック

目的と役割に応じて、モダンかつ検証・保守しやすい技術を選定する。

### 1. フロントエンド (ユーザーインターフェース)
- **コア:** `React` + `TypeScript` + `Vite` (高速なSPA開発基盤)
- **状態管理:** `Zustand` (リアルタイムプレビューを支える軽量・高パフォーマンスな状態管理)
- **UI / スタイリング:** `shadcn/ui` + `TailwindCSS` (コンポーネント駆動の高速なデザインシステム)
- **品質管理:** `ESLint` + `Prettier`

### 2. バックエンド (API ＆ 帳票変換・生成エンジン)
- **フレームワーク:** `Python` + `FastAPI` (型安全かつ高速な非同期API開発)
- **PDF解析・変換:** `Docling` (既存PDFのレイアウト・テキストの高度な解析)
- **生成AI統合:** `Anthropic Python SDK (Claude API)` (保守性の高いHTML/CSS構造の生成)
- **ORM / データベース接続:** `SQLAlchemy` (将来的なDB保存を見据えた抽象化)

### 3. テスト ＆ 品質保証 (TDDの強固な基盤)
- **フロントエンドテスト:** `Vitest` + `React Testing Library` (高速なユニット/コンポーネントテスト)
- **バックエンドテスト:** `pytest` (柔軟かつ強力なAPI・ロジックテスト)
- **ブラウザ自動化 (E2E):** `Playwright` (一連のユーザー行動と描画結果の自動検証)

### 4. インフラ ＆ CI/CD / 外部サービス
- **認証・認可・データベース:** `Supabase` (Auth + PostgreSQLをクラウド管理、開発用の `Supabase Local CLI` 併用)
- **IaC (インフラコード化):** `Terraform` (AWSおよびSupabaseプロバイダーの一元管理)
- **インフラ（AWS）:** `CloudFront` + `S3` (フロントエンド静的配信), `AWS Lambda (Docker)` + `API Gateway` (サーバーレスバックエンド), `AWS WAF` (セキュリティ防御)
- **CI/CD:** `GitHub Actions` (テスト自動化およびAWSデプロイ)

### 5. 開発環境
- **AI共同開発ツール:** `ClaudeCode`

## 🎨 フロントエンド要件

### 🗺️ 画面構成要素
1. **HTMLプレビュー表示エリア** (レスポンシブ・コンポーネント)
2. **4大入力エディタ:** HTML入力 / CSS入力 / JSON入力 / プロンプト入力エリア
3. **ファイル操作:** PDFアップロードエリア (ドラッグ＆ドロップ対応)
4. **コントロール:** 縦幅・横幅サイズ入力項目、描画ボタン (ピクトグラム)
5. **アーキテクチャインフォメーション (インライン表示 / Mermaid埋め込み):**
   - バックエンドAPI設計概要図 / システム構成図 / セキュリティ概要図 / CI-CD概要図

### ⚙️ 主要機能
- **リアルタイム双方向プレビュー**
  - HTML/CSS/JSONの変更をリアルタイムに検知し、帳票プレビューに即座に反映。
  - 画面幅に合わせたアスペクト比固定スケーリング（PC/iPhoneそれぞれで最低/最大固定サイズを設定）。
- **インテリジェントテンプレート連動**
  - HTML内のテンプレート変数とJSONのキー情報を柔軟に連携・マッピング。
  - タイトル等の固定情報はHTMLに直接記述し、明細データ等の業務データのみをJSONと連動。
- **定型サイズ自動入力**
  - A5/B5/A4（それぞれ「たて」「よこ」）を選択すると、対応する縦横の寸法が自動入力される。
- **履歴スライド機能**
  - 描画ボタン押下時、全コンテキスト（PDF, HTML, CSS, プロンプト, JSON, サイズ）をAPIへ送信。
  - レスポンス（HTML/CSS/JSON）を反映。再描画時は、過去の描画内容が最大10件まで横にスライドしてスタックされる。
- **インテリジェントメッセージ表示**
  - バックエンドAPIのステータスコード（4xx, 5xx等）に完全準拠したエラー/成功メッセージをトーストやトレイで分かりやすく表示。

## 🧠 バックエンド要件 (AWS Lambda + Python)

### 🚀 ロジックフロー
1. **リクエスト受領:** `[PDF, JSON, CSS, プロンプト, 縦幅, 横幅]` を受領。
2. **PDF解析 (Docling):** PDFが存在する場合、`docling` を用いて事前に高精度なベースHTML/CSSへ変換。
3. **動的プロンプト構築:** 送信されてきた要素の有無（PDF/CSS/JSON/プロンプト/サイズ等の有無）に応じて、Claudeへの指示プロンプトを動的に最適化。
4. **Claudeによる構造化生成:**
   - エンジニアが保守しやすいクリーンなHTML/CSS/JSONを生成させる。
   - 固定テキストはHTMLへ、動的な業務データはテンプレート変数へ明確に分離。
5. **構造化レスポンス:** 生成されたHTML, CSS, JSONを構造化（JSON形式）して返却。

### 🚨 エラーハンドリング
- バリデーションエラー、AI生成エラー、Docling解析エラーなど、例外や内容に応じたHTTPステータスコードを厳格に定義してレスポンスする。

## 💡 実務的な開発クオリティの担保（落とし穴への対策）

1. **型安全の自動担保（型同期スキーム）**
   - フロントエンドとバックエンドの疎通における「キー名のズレ」を防ぐため、FastAPIが自動生成する `openapi.json` からフロント用のTypeScript型定義を自動生成する仕組みを導入する。
2. **AI課金コスト・速度への対策（モック戦略）**
   - 開発時や pytest 実行時に毎回 Anthropic API を叩くとコストとオーバーヘッドが発生するため、テスト環境では「プロンプトに応じた疑似HTML/CSS/JSON」を即座に返す強固なモックレイヤーを実装する。
3. **Doclingのローカル依存対策**
   - DoclingはOS依存のバイナリや機械学習モデルを内包するため、開発初期段階で単体での検証スクリプトを動かし、環境依存のトラブルを早期に解消する。
4. **環境を汚さないローカルDB検証**
   - フェーズ5でSupabaseを導入する際、いきなりクラウド環境に依存せず、Supabase Local CLIやDockerを用いてローカルでマイグレーションやデータ検証、テストが完結する環境を構築する。

## 🔒 セキュリティ & アクセス設計

### 🌐 1. 自由アクセス（未認証）エリア
- **認証認可:** 不要
- **アクセス制御:** AWS WAF等によるIP制限、および過度なAPIコールを防ぐ厳格なレート制限。
- **データ制約:** データベース等の機密情報へのアクセスは一切不可（ステートレスな変換・生成のみ）。

### 🔐 2. アカウント登録（認証あり）エリア
- **認証認可:** **Supabase Auth** によるセキュアなユーザー認証およびJWTによる認可。
- **アクセス制御:** アカウント（ユーザーID）ベースのレート制限。
- **データ制約:** データベース（Supabase）へのアクセスを許可し、ユーザー固有の帳票テンプレートや生成履歴などの機密情報の保存・閲覧を許可。
